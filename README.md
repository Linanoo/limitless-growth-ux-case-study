# limitless-growth-ux-case-study

🌱 Limitless Growth – Mobile Garden Planning App

UX Research | UI Design | Prototyping | Usability Testing

📌 Project Overview

Limitless Growth is a mobile-first garden planning application designed to help hobby gardeners efficiently plan, manage, and optimise their vegetable harvest.

The primary goal of the project was to reduce cognitive load and planning time while improving harvest consistency through structured guidance, reminders, and collaborative features.

This project was developed as part of an HCI module at the University of Sussex and followed a full user-centred design process:

Persona Development

Context Scenario Design

Competitive Research

Low-Fidelity Prototyping

Usability Testing

Iterative Refinement

🎯 Problem Statement

Many hobby gardeners:

Spend excessive time planning seasonal planting

Struggle to diagnose plant issues

Forget sowing and harvesting timelines

Cannot distinguish between beneficial and harmful garden critters

Experience inconsistent vegetable yields year-to-year

Lack a single trusted source of gardening information

The objective was to design an intuitive mobile application that centralises garden management while remaining accessible for users with low-to-medium technical confidence.

👤 Primary Persona

Ben Lowen
Age: 58
Location: Stoke-on-Trent
Technical Comfort: Low–Medium

Key Characteristics:

Hardware engineer

Enjoys gardening but finds planning tedious

Wants to maximise harvest yield

Struggles with misinformation online

Needs accessibility-friendly UI (larger text, clarity, easy reach zones)

Core Goals:

Minimise time spent planning

Maximise vegetable output

Track sowing and harvesting times

Diagnose plant problems easily

Collaborate with community gardeners

🔎 User Research & Insights

Interviews and competitive analysis revealed:

Expectations from a garden planning app:

Location-based planting guidance

Seasonal reminders

Pest & disease diagnosis support

Community Q&A functionality

Accessibility features (text size, dark mode)

Clear and minimal interface

Competitive Observations:

Many apps rely heavily on AI chatbots (trust concerns)

Some dashboards are overly cluttered

Region-locked data limits usefulness

Poor web support

Inconsistent design language across screens

This informed a strong emphasis on clarity, usability, and consistency.

🧠 Design Requirements

The app needed to support:

Taking structured notes when receiving gardening advice

Sharing plants and images with a community

Forum-based question & answer system

Diagnosing dying plants

Identifying beneficial vs harmful critters

Sowing at optimal seasonal timing

Harvest reminders

Accessibility-first interface

Garden collaboration

Ask gardening questions

⚙️ Accessibility

Clear typography

High contrast colour scheme

Minimal cognitive load layout

Scalable interface considerations

🎨 Design Approach

The visual direction prioritised:

Minimalistic green-based colour palette

Large, readable typography

Clear visual hierarchy

Thumb-zone ergonomics

Calm and supportive aesthetic

Consistency and clarity were key themes across all screens.

🧪 Usability Testing

Low-fidelity prototype testing identified several usability issues.

Key Findings:

Floating action button (Add Plant) unclear in function

Accidental pressing of floating button while scrolling

No multi-select functionality in plant management

Inconsistent component widths across screens

Scaling issues across device sizes

Confusion when managing multiple gardens

Grow box feature unclear in purpose

🔄 Iterative Improvements

Based on testing, the following refinements were proposed:

Onboarding tutorial explaining key actions

Multi-select plant management

Fixed-width layout system

Device-size responsiveness testing

Colour-coded garden system to prevent cross-management errors

Relocation and clearer explanation of Grow Box

Improved icon labelling on Critters page

This iterative approach demonstrated structured UX refinement rather than static design.

📊 Outcome

The prototype:

Successfully met core user goals

Improved usability for beginner gardeners

Provided structured knowledge access

Introduced collaborative features

However:

Further testing with multiple participants is required

Advanced gardeners may require more depth

Community features require expansion to compete with existing platforms
